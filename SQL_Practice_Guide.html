Basics

What is SQL · Syntax · Data Types · SELECT · DISTINCT · Aliases (AS)
🧒 Explain Like I'm 5
🇬🇧 English
Imagine you have a big box of toy cards — each card has a name, age, and colour. SQL is like a magic question you ask the box: "Show me all the cards!" The database is that box, and a table is like one section inside it. SELECT means "show me", and FROM means "from this box".
🇮🇳 தமிழ்
கற்பனை செய்து பாரு — உன்னிடம் ஒரு பெரிய பெட்டி இருக்கு, அதில் நிறைய அட்டைகள் இருக்கு (ஒவ்வொரு அட்டையிலும் பெயர், வயது, நிறம் இருக்கு). SQL என்பது அந்த பெட்டியிடம் கேட்கும் ஒரு மாயாஜால கேள்வி: "எல்லா அட்டைகளையும் காட்டு!" Database என்பது அந்த பெட்டி; Table என்பது பெட்டியின் ஒரு பகுதி. SELECT = "காட்டு", FROM = "இந்த பெட்டியிலிருந்து".
📝 Sample Query & Output
-- Get all patients
SELECT * FROM patients;

-- Get only name and age (specific columns)
SELECT name, age FROM patients;

-- DISTINCT: unique cities only
SELECT DISTINCT city FROM patients;

-- AS: rename column in output
SELECT name AS patient_name, age AS years_old FROM patients;

Filtering Data

WHERE · Comparison Operators · AND/OR/NOT · IN · BETWEEN · LIKE · IS NULL
🧒 Explain Like I'm 5
🇬🇧 English
You have 100 toy cards. You only want the red ones. You use a filter! In SQL, WHERE is that filter. It says: "Only show me rows that match this rule." Like: only show patients from Chennai, or only doctors who earn more than $5000.
🇮🇳 தமிழ்
உன்னிடம் 100 அட்டைகள் இருக்கு. நீ சிவப்பு நிற அட்டைகள் மட்டும் வேணும்னு சொல்றே. அதுக்கு ஒரு வடிகட்டி (filter) பயன்படுத்துவோம். SQL-ல அந்த வடிகட்டி தான் WHERE. "இந்த நிபந்தனைக்கு பொருந்தும் rows மட்டும் காட்டு" என்று சொல்ல பயன்படுகிறது. உதாரணம்: Chennai-லிருந்து வரும் நோயாளிகள் மட்டும், அல்லது 5000-க்கு மேல் சம்பளம் பெறும் டாக்டர்கள் மட்டும்.
📝 Sample Query & Output
-- WHERE with comparison
SELECT name, age FROM patients WHERE age > 40;

-- AND / OR
SELECT * FROM patients WHERE city = 'Chennai' AND gender = 'Male';

-- IN operator
SELECT name FROM patients WHERE city IN ('Chennai', 'Mumbai');

-- BETWEEN
SELECT name, age FROM patients WHERE age BETWEEN 30 AND 50;

-- LIKE: patients whose name starts with 'A'
SELECT name FROM patients WHERE name LIKE 'A%';

-- IS NULL: appointments with no fee
SELECT * FROM appointments WHERE fee IS NULL;
▶ Output: WHERE age > 40
nameage
Ramesh Babu52
Kavitha Nair45

Sorting & Limiting

ORDER BY · ASC / DESC · LIMIT / TOP
🧒 Explain Like I'm 5
🇬🇧 English
Imagine you have a list of scores. You want to arrange them from highest to lowest — that's ORDER BY DESC. And if you only want to see the top 3, you say LIMIT 3. Easy!
🇮🇳 தமிழ்
உன்னிடம் மதிப்பெண்கள் பட்டியல் இருக்கு. அதை அதிகமானதிலிருந்து குறைவானது வரை வரிசைப்படுத்தணும் — அதுதான் ORDER BY DESC. முதல் 3 பேர் மட்டும் வேணும்னா LIMIT 3 சொல்லலாம். மிகவும் எளிது!
📝 Sample Query & Output
-- Sort patients by age (oldest first)
SELECT name, age FROM patients ORDER BY age DESC;

-- Top 3 highest-paid doctors
SELECT name, salary FROM doctors ORDER BY salary DESC LIMIT 3;

-- Sort appointments by date (most recent first)
SELECT appt_id, appt_date, status FROM appointments
ORDER BY appt_date DESC;

Aggregate Functions

COUNT · SUM · AVG · MIN · MAX · ROUND
🧒 Explain Like I'm 5
🇬🇧 English
You have 10 piggy banks with different amounts of money. SUM adds them all up. AVG finds the average. COUNT counts how many banks there are. MAX finds the richest bank. These are your math helpers in SQL!
🇮🇳 தமிழ்
உன்னிடம் 10 குட்டி பணப்பெட்டிகள் இருக்கு, ஒவ்வொன்றிலும் வெவ்வேறு தொகை இருக்கு. SUM எல்லாவற்றையும் கூட்டும். AVG சராசரி காண்பிக்கும். COUNT எத்தனை பெட்டிகள் என்று எண்ணும். MAX அதிகமான தொகையை காண்பிக்கும். இவை SQL-ல உனது கணித உதவியாளர்கள்!
📝 Sample Query & Output
-- Count total patients
SELECT COUNT(*) AS total_patients FROM patients;

-- Total revenue from appointments
SELECT SUM(fee) AS total_revenue FROM appointments;

-- Average doctor salary
SELECT ROUND(AVG(salary), 2) AS avg_salary FROM doctors;

-- Min and Max appointment fee
SELECT MIN(fee) AS min_fee, MAX(fee) AS max_fee FROM appointments;
▶ Output

Grouping Data

GROUP BY · HAVING · GROUP BY Multiple Columns · Aggregate + GROUP BY
🧒 Explain Like I'm 5
🇬🇧 English
Imagine sorting your toys into groups: all cars together, all dolls together, all blocks together. Then you count how many toys are in each group. That's GROUP BY! And HAVING is like saying "only show me groups that have more than 5 toys."
🇮🇳 தமிழ்
உன் பொம்மைகளை பிரிக்கிறாய்: கார்கள் ஒரு குழு, பொம்மைகள் ஒரு குழு, தொகுதிகள் ஒரு குழு. பிறகு ஒவ்வொரு குழுவிலும் எத்தனை பொம்மைகள் என்று எண்ணுகிறாய். அதுதான் GROUP BY! HAVING என்பது "5-க்கும் மேல் பொம்மைகள் உள்ள குழுக்களை மட்டும் காட்டு" என்று சொல்வது போன்றது.
📝 Sample Query & Output
-- Count patients per city
SELECT city, COUNT(*) AS patient_count
FROM patients
GROUP BY city;

-- Total fees collected per doctor
SELECT doctor_id, SUM(fee) AS total_fees
FROM appointments
GROUP BY doctor_id;

-- HAVING: only cities with more than 5 patients
SELECT city, COUNT(*) AS patient_count
FROM patients
GROUP BY city
HAVING COUNT(*) > 5;

Joins

INNER JOIN · LEFT JOIN · RIGHT JOIN · FULL JOIN · CROSS JOIN · SELF JOIN
🧒 Explain Like I'm 5
🇬🇧 English
You have two lists: one with student names, another with their exam scores. To see "Arjun's score", you need to join the two lists by matching the student name. INNER JOIN shows only students who appear in BOTH lists. LEFT JOIN shows everyone from the first list, even if they have no score.
🇮🇳 தமிழ்
உன்னிடம் இரண்டு பட்டியல்கள் இருக்கு: ஒன்று மாணவர் பெயர்கள், மற்றொன்று அவர்களின் மதிப்பெண்கள். "அர்ஜுன்னின் மதிப்பெண்" பார்க்க, இரண்டு பட்டியல்களையும் இணைக்க வேண்டும். INNER JOIN = இரண்டு பட்டியலிலும் இருப்பவர்களை மட்டும் காட்டும். LEFT JOIN = முதல் பட்டியலில் உள்ள எல்லோரையும் காட்டும், மதிப்பெண் இல்லாதவர்களையும் சேர்த்து.
📝 Sample Query & Output
-- INNER JOIN: appointments with patient names
SELECT a.appt_id, p.name AS patient, a.appt_date, a.fee
FROM appointments a
INNER JOIN patients p ON a.patient_id = p.patient_id;

-- LEFT JOIN: all patients, even without appointments
SELECT p.name, a.appt_date
FROM patients p
LEFT JOIN appointments a ON p.patient_id = a.patient_id;

-- 3-table JOIN: appointment with patient AND doctor names
SELECT p.name AS patient, d.name AS doctor,
       d.department, a.appt_date, a.fee
FROM appointments a
INNER JOIN patients p ON a.patient_id = p.patient_id
INNER JOIN doctors d ON a.doctor_id = d.doctor_id;
▶ Output: 3-table JOIN (sample rows)

Subqueries

Subquery in WHERE · Subquery in SELECT · Subquery in FROM · Correlated · EXISTS
🧒 Explain Like I'm 5
🇬🇧 English
A subquery is a question inside a question! Like asking: "Show me students who scored higher than (the average score of the whole class)." The part in brackets is the inner question — it runs first, then the outer question uses that answer.
🇮🇳 தமிழ்
ஒரு கேள்விக்குள் இன்னொரு கேள்வி! உதாரணம்: "வகுப்பின் சராசரி மதிப்பெண்ணை விட அதிகம் பெற்ற மாணவர்களை காட்டு." அடைப்புக்குறிக்குள் உள்ளது உள் கேள்வி — அது முதலில் இயங்கும், பிறகு வெளி கேள்வி அந்த விடையை பயன்படுத்தும்.
📝 Sample Query & Output
-- Subquery in WHERE: doctors earning above average salary
SELECT name, salary FROM doctors
WHERE salary > (SELECT AVG(salary) FROM doctors);

-- Subquery in SELECT: show each patient + total their appointments
SELECT name,
  (SELECT COUNT(*) FROM appointments a
   WHERE a.patient_id = p.patient_id) AS total_appointments
FROM patients p;

-- EXISTS: patients who have at least one appointment
SELECT name FROM patients p
WHERE EXISTS (
  SELECT 1 FROM appointments a
  WHERE a.patient_id = p.patient_id
);

Set Operators

UNION · UNION ALL · INTERSECT · EXCEPT / MINUS
🧒 Explain Like I'm 5
🇬🇧 English
You have two friend groups. UNION combines both groups (no duplicates). UNION ALL combines them but keeps duplicates. INTERSECT shows only friends in BOTH groups. EXCEPT shows friends only in group 1 but NOT in group 2.
🇮🇳 தமிழ்
உன்னிடம் இரண்டு நண்பர் குழுக்கள் இருக்கு. UNION = இரண்டையும் இணைக்கும் (duplicate இல்லாமல்). UNION ALL = இரண்டையும் இணைக்கும் (duplicate உடன்). INTERSECT = இரண்டு குழுவிலும் இருக்கும் நண்பர்கள். EXCEPT = முதல் குழுவில் மட்டும் இருக்கும் நண்பர்கள்.
📝 Sample Query & Output
-- UNION: all cities from patients + all locations from departments
SELECT city AS location FROM patients
UNION
SELECT location FROM departments;

-- INTERSECT: patients who are also from department locations
SELECT city FROM patients
INTERSECT
SELECT location FROM departments;

-- EXCEPT: patient cities NOT covered by any department
SELECT city FROM patients
EXCEPT
SELECT location FROM departments;
❓ Practice Questions

✅ Answer

UNION

Window Functions

OVER() · ROW_NUMBER() · RANK() · DENSE_RANK() · LAG() · LEAD() · PARTITION BY
🧒 Explain Like I'm 5
🇬🇧 English
Imagine you have a class, and you want to rank each student's score — but you want to keep all the rows (unlike GROUP BY which squishes them). Window functions do calculations across rows without collapsing them. RANK() gives each row a rank number. LAG() lets you peek at the previous row!
🇮🇳 தமிழ்
வகுப்பில் உள்ள மாணவர்களுக்கு rank கொடுக்கணும், ஆனால் எல்லா rows-ஐயும் வைத்திருக்கணும் (GROUP BY போல அழிக்கக்கூடாது). Window functions எல்லா rows-ஐயும் வைத்திருந்தே கணக்கு போடும். RANK() = ஒவ்வொரு row-க்கும் rank கொடுக்கும். LAG() = முந்தைய row-ஐ பார்க்க உதவும்!
📝 Sample Query & Output
-- ROW_NUMBER: assign a unique row number
SELECT name, salary,
  ROW_NUMBER() OVER (ORDER BY salary DESC) AS row_num
FROM doctors;

-- RANK within each department
SELECT name, department, salary,
  RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dept_rank
FROM doctors;

-- LAG: compare each appointment fee with the previous one
SELECT appt_id, appt_date, fee,
  LAG(fee) OVER (ORDER BY appt_date) AS prev_fee
FROM appointments;

For each appointment, show the fee difference from the previous appointment using LAG().
✅ Answer
SELECT appt_id, appt_date, fee,
  LAG(fee, 1) OVER (ORDER BY appt_date) AS prev_fee,
  fee - LAG(fee, 1) OVER (ORDER BY appt_date) AS fee_diff
FROM appointments;
Q3
Find the top-ranked doctor in each department by salary (PARTITION BY + RANK).
✅ Answer
SELECT * FROM (
  SELECT name, department, salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rnk
  FROM doctors
) ranked
WHERE rnk = 1;

-- CASE: categorise patients by age
SELECT name, age,
  CASE
    WHEN age < 18 THEN 'Child'
    WHEN age BETWEEN 18 AND 60 THEN 'Adult'
    ELSE 'Senior'
  END AS age_group
FROM patients;

-- CTE: top-earning doctors per department
WITH DeptMax AS (
  SELECT department, MAX(salary) AS max_salary
  FROM doctors
  GROUP BY department
)
SELECT d.name, d.department, d.salary
FROM doctors d
JOIN DeptMax dm ON d.department = dm.department
  AND d.salary = dm.max_salary;

-- VIEW: create a reusable appointment summary
CREATE VIEW appointment_summary AS
SELECT p.name AS patient, d.name AS doctor,
       d.department, a.appt_date, a.fee, a.status
FROM appointments a
JOIN patients p ON a.patient_id = p.patient_id
JOIN doctors d ON a.doctor_id = d.doctor_id;

-- Conditional Aggregation: count completed vs pending per doctor
SELECT doctor_id,
  COUNT(CASE WHEN status = 'Completed' THEN 1 END) AS completed,
  COUNT(CASE WHEN status = 'Pending' THEN 1 END) AS pending
FROM appointments
GROUP BY doctor_id;

Write a CASE statement to label appointment fees as 'Low' (<1000), 'Medium' (1000–3000), or 'High' (>3000).

SELECT appt_id, fee,
  CASE
    WHEN fee < 1000 THEN 'Low'

Q2
Write a CTE to first get total fees per doctor, then show only those who collected more than 50,000.
✅ Answer
WITH DoctorRevenue AS (
  SELECT doctor_id, SUM(fee) AS total_revenue
  FROM appointments
  GROUP BY doctor_id
)
SELECT d.name, dr.total_revenue
FROM DoctorRevenue dr
JOIN doctors d ON dr.doctor_id = d.doctor_id
WHERE dr.total_revenue > 50000
ORDER BY dr.total_revenue DESC;
Q3
Create a VIEW called active_patients that shows patients who have had at least one appointment in the last year.
✅ Answer
CREATE VIEW active_patients AS
SELECT DISTINCT p.patient_id, p.name, p.city
FROM patients p
INNER JOIN appointments a ON p.patient_id = a.patient_id
WHERE a.appt_date >= DATE_SUB(CURDATE(), INTERVAL 1 YEAR);
